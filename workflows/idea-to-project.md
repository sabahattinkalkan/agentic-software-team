# Idea to Project Workflow

This document defines the **canonical execution flow**
for turning a raw product idea into a review-ready software project
using the Agentic Software Team.

---

## Step 0 — Input

**Input:**

- Raw product idea (1–3 sentences)
- Optional: target audience, constraints, priorities

This input is always given to the **Team Lead Agent** first.

---

## Step 1 — Team Lead Analysis

**Agent:** Team Lead

**Responsibilities:**

- Analyze the idea
- Define scope (MVP vs extended)
- Break down the project
- Decide which agents will be involved

**Output:**

- Project breakdown
- Execution plan
- Risk notes

**Next:** CTO Agent

---

## Step 2 — Technical Direction

**Agent:** CTO

**Responsibilities:**

- Validate scope technically
- Propose architecture
- Select technology stack
- Define technical guardrails

**Output:**

- Architecture proposal
- Tech stack
- Technical risks

**Next:** UX Designer

---

## Step 3 — User Experience Design

**Agent:** UX Designer

**Responsibilities:**

- Define user personas
- Map core user flows
- Reduce unnecessary scope
- Identify UX risks

**Output:**

- User personas
- Core user flows
- Scope reduction notes

**Next:** UI Designer

---

## Step 4 — User Interface System

**Agent:** UI Designer

**Responsibilities:**

- Define design system
- Translate UX flows into interface structures
- Provide frontend guidance

**Output:**

- Design system
- Component inventory
- UI implementation notes

**Next:** Backend Developer

---

## Step 5 — Backend Definition

**Agent:** Backend Developer

**Responsibilities:**

- Define data models
- Design APIs
- Implement business logic concepts
- Address security concerns

**Output:**

- Backend architecture notes
- Data models
- API definitions

**Next:** Frontend Developer

---

## Step 6 — Frontend Definition

**Agent:** Frontend Developer

**Responsibilities:**

- Define component structure
- Plan state management
- Integrate with backend APIs
- Address accessibility

**Output:**

- Component hierarchy
- State management plan
- Integration notes

**Next:** QA / Bug Hunter

---

## Step 7 — Quality Assurance

**Agent:** QA / Bug Hunter

**Responsibilities:**

- Test flows and assumptions
- Identify bugs and edge cases
- Validate error handling and permissions

**Output:**

- Bug list
- Edge-case inventory
- Quality risk notes

**Next:** Marketing Strategist

---

## Step 8 — Marketing Strategy

**Agent:** Marketing Strategist

**Responsibilities:**

- Define target market
- Clarify value proposition
- Position the product
- Prepare go-to-market logic

**Output:**

- Market definition
- Value proposition
- Core messaging

**Next:** Ads & Growth Agent

---

## Step 9 — Growth & Acquisition

**Agent:** Ads & Growth

**Responsibilities:**

- Define acquisition funnels
- Plan paid channels
- Define metrics and experiments

**Output:**

- Acquisition funnel
- Growth metrics
- Optimization plan

**Next:** Review & Audit Agent

---

## Step 10 — Final Review

**Agent:** Review & Audit

**Responsibilities:**

- Cross-check all outputs
- Identify contradictions and gaps
- Assess readiness
- Issue go / no-go decision

**Output:**

- Consistency report
- Risk & gap analysis
- Final verdict

---

## Completion

A project is considered **ready** when:

- Review & Audit Agent issues a **Go** or **Conditional Go**
- All mandatory fixes are identified
